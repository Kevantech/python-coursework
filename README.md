# Black-Box Optimisation Capstone Project

## NON-TECHNICAL EXPLANATION OF YOUR PROJECT

Imagine trying to find the highest point in a mountain range while blindfolded — you can only ask "how high is it here?" at a limited number of spots. This project tackles exactly that challenge: optimising a set of unknown ("black-box") functions where I can see what output each input produces, but never how the function works inside. Using machine learning — mainly Bayesian optimisation, which builds a statistical "map of belief" from past queries — I made a limited budget of queries count, balancing exploration of unknown regions with exploitation of promising ones to find the best possible inputs.

## DATA

The "data" in this project is generated by me, one query at a time. Each week I submitted input points to a set of hidden black-box functions through the capstone project portal and received back a single numerical output per function. The functions have different input dimensions (from 2 up to 8 dimensions), and all inputs lie in a bounded, continuous domain. No external or public dataset is used; the accumulated table of (input → output) pairs from my own queries is the entire dataset, and it grows with every submission round. All query histories are stored in this repository as CSV files in the `data/` folder.

## MODEL

My main approach is **Bayesian optimisation with a Gaussian process (GP) surrogate model**. A Gaussian process is well suited to this problem because it works with very few data points, provides not just a prediction but also an uncertainty estimate at every candidate input, and makes no strong assumptions about the shape of the hidden function. On top of the GP, an **acquisition function** (Expected Improvement, with Upper Confidence Bound as a comparison) decides where to query next by trading off high predicted values against high uncertainty.

For the initial rounds, before enough data existed to fit a useful surrogate, I used **space-filling exploration** (random/Sobol sampling) to cover the input domain evenly. For the higher-dimensional functions I also compared the GP approach against a **random forest surrogate**, which scales more comfortably as dimensions grow.

## HYPERPARAMETER OPTIMISATION

The key hyperparameters in this pipeline are:

- **GP kernel choice** (RBF vs Matérn 5/2) and its **length-scale** — the length-scale was fitted by maximising the marginal log-likelihood on the observed queries each round; Matérn 5/2 was preferred as it handles less-smooth functions better.
- **Acquisition function and its exploration parameter** — Expected Improvement's ξ (xi) and UCB's κ (kappa) control how adventurous the next query is. I started with high exploration (large κ) in early rounds and reduced it in later rounds as the surrogate became more confident.
- **Number of initial random points** before switching to model-guided queries — chosen per function relative to its dimensionality (more dimensions → more initial coverage).

Because the true objective is hidden, hyperparameters could not be tuned against ground truth. Instead, I evaluated them by how much each new query improved the best-observed value round over round, and by cross-validating the surrogate's predictions on held-out past queries.

## RESULTS

Across the query rounds, the best-observed value for each function improved steadily, with the largest gains coming right after switching from random exploration to Bayesian optimisation. The 2D function surface reconstructed from my queries is shown below — the model correctly identified the high-value region (bright yellow, upper-right area) and concentrated later queries there, while the early space-filling queries mapped out the overall landscape.

![Screenshot](image.png)

The main lessons from this project: (1) uncertainty estimates are as valuable as predictions when data is scarce — they tell you where you *don't* know enough; (2) the exploration–exploitation balance matters more than the choice of surrogate model in the early rounds; and (3) higher-dimensional functions need disproportionately more exploration, which makes budget allocation across functions a strategic decision in itself.
